

Race Conditions is a situation where
- Two or more threads access the same variables in a way where the final result
    stored in variable depends on how thread access to variables are scheduled.

Race Conditon occur when-

-   Two or more thread read and write the same variable or data concurrently
-   The thread access the variable using either pattern:
    -   Check then act
    -   Read Modify write
        -   Where modified value depends on previosly read value
    - The thread access to the variables is not atomic
